## メイン／寄り道／征討（P3-D5DG-002 / P3-LORE-006）

| 種別 | `route_type` | 役割 | 規模目安 | 進行 |
|---|---|---|---|---|
| **メイン** | `main` | ストーリー脊柱・九王フック・次帯解放の鍵 | **5 章 / Biome**（1-1〜1-5）。最終章 floor 8〜10＋末尾 Boss | 最終章 Boss **初回討伐**（ノーマル）で次メイン **第1章** 解放 |
| **寄り道** | `side` | 属性素材・戦力整備・地理の枝道 | 雑魚2〜3＋Elite0〜1・floor 5〜7 | **必須ゲートにしない**（`unlock_after_dungeon_id`） |
| **征討** | `apex` | メイン Boss 超えの伝説個体（Class VII） | floor 7〜10・征討 Boss1 | 紐づく帯クリアで解放。**次メイン解放に影響しない** |

**実装済（18本）:** SSOT=`world/07_Geography §4.1`・征討 Boss=`04_Classification §5.6`。**プレイ対象はメイン5本のみ**（寄り道・征討は **P3-DG-OMIT-001 で一旦オミット**・データは残置）。

**難易度の置き方:** 新メイン初挑戦は、前帯メイン＋寄り道の周回装備が無いと **敗北 or ギリギリ** になりうる数値にする。クリア後は周回トグル（P3-D118/D142）でファーム効率化。

**データ:** `DungeonData.route_type` 実装済。`parent_biome_id`・`recommended_runs_before_main` は将来 Task。

## 階層数・イベント（P3-D5DG-003）

| 項目 | 方針 |
|---|---|
| `floor_count` | **章ごとに個別**（1-1〜1-4 は可変・**1-5 は 8〜10＋末尾 Boss**）。**1 章 5〜10 分**（P3-DG-STG-001）を優先 |

## 部屋進行（MVP実装）

### 固定シーケンス（branch_enabled = false）

`DungeonController.ROOM_SEQUENCE` に従う 10 部屋構成。

```text
START → COMBAT → EVENT → TREASURE → ELITE → EVENT → COMBAT → MID_BOSS → BOSS → EXIT
```

`branch_enabled = false` のダンジョンは上記固定順。`mourngate.tres` は `branch_enabled = true`（下記 Branch Route）。

### Branch Route System（Phase2-M3 実装）

`DungeonData.branch_enabled = true` のダンジョンで有効。

**分岐タイミング:** 次の部屋が MID_BOSS / BOSS / EXIT 以外のとき、部屋進行前に分岐選択 UI を表示。

**選択肢（BranchContainer）:**

| 選択 | 内部値 | プール |
|---|---|---|
| 安全ルート | 0 | SAFE_POOL |
| 危険ルート | 1 | DANGEROUS_POOL |
| 不明ルート | 2 | UNKNOWN_POOL |

**v3.4+ 確定プール（SSOT）:**

| Pool | 内容 |
|---|---|
| SAFE_POOL | HEAL / TREASURE |
| DANGEROUS_POOL | COMBAT / ELITE |
| UNKNOWN_POOL | COMBAT / HEAL / EVENT / TREASURE |

各プールからランダムに 1 部屋タイプを選び `_next_room_type` に設定。`advance_room()` 時に適用。

**固定部屋:** MID_BOSS / BOSS / EXIT は `ROOM_SEQUENCE` 固定。分岐選択なし。

## 部屋タイプ（Phase2-M3 実装分）

### HEAL Room（回復の部屋）

| 項目 | 値 |
|---|---|
| RoomType | `Enums.RoomType.HEAL` |
| 回復量 | 10 HP（`DungeonScene.HEAL_AMOUNT`） |
| 対象 | 生存メンバーのみ |
| 処理 | `CombatController.heal_party(10)` — 最大 HP を超えない |

戦闘部屋ではない。入室ログ: 「回復の部屋: 生存メンバーを10回復」

### TREASURE Room（宝箱の部屋）

| 項目 | 値 |
|---|---|
| RoomType | `Enums.RoomType.TREASURE` |
| Gold | +30（`TREASURE_GOLD`） |
| 装飾品 | 20% で silver_ring 抽選（`TREASURE_ACCESSORY_CHANCE`） |
| 処理 | `DungeonController.generate_treasure_loot()` |

報酬は `run_gold_reward` に累積。装飾品は `GameState.accessory_inventory` に追加。

### MERCHANT Room — **削除（P3-D070）**

商人部屋は **P3-D070 で削除済み**。`Enums.RoomType.MERCHANT` は列挙に残るが、`ROOM_SEQUENCE` / Branch Route プールからは除外。関連 UI・カタログ・購入ハンドラは除去。

### EVENT Room（イベントの部屋）— P3-D070 準拠

Phase2-M3 Special Room。**戦闘なし・プレイヤー選択なし**（フルオート）。

| 項目 | 値 |
|---|---|
| RoomType | `Enums.RoomType.EVENT` |
| 出現 | UNKNOWN_POOL（Branch Route）/ 固定シーケンス（index 2, 5） |
| 解決 | `auto_resolve_event()` で単一 `outcome` を即時適用 |
| 所要時間 | 数秒（説明表示→結果ログ→自動進行） |

**報酬カテゴリ:**

| type | イベント例 | 効果 |
|---|---|---|
| heal | 崩れた祭壇 | 生存メンバー +8 HP |
| gold | 古文書 | run 報酬 Gold +25 |
| buff | 封印された扉 | 周回内攻撃 x1.15（`run_damage_multiplier`） |
| material | 朽ちた木箱 | `relic_shard` x1 → `material_inventory` |
| lore | 色あせた碑文 | 図鑑登録（`discovery_id`） |

**操作フロー:**

1. 入室 → `pick_event()` で 汎用 `EVENTS` + `DUNGEON_EVENTS[dungeon_id]` を結合してランダム 1 件
2. `description`（宣言文）をログ表示 → `auto_resolve_event()` で結果適用
3. `_start_auto_progress()` で次の部屋へ自動進行

**UI:** 選択ボタンなし。`EventContainer` は未使用（将来バナー表示用に `BaseScene.EventBanner` がプレースホルダー）。

**設計意図:** 探索の多様性を数秒で追加しつつ、フルオートのテンポを維持。

### ELITE Room（エリートの部屋）— P2-Task017 確定

Phase2-M3 Special Room。**常に戦闘**。高リスク・高報酬。

| 項目 | 値 |
|---|---|
| RoomType | `Enums.RoomType.ELITE` |
| 出現 | DANGEROUS_POOL（Branch Route）/ 固定シーケンス（index 4） |
| 敵選択 | `pick_elite_enemy_data()` → `DungeonData.elite_pool` |
| フォールバック | elite_pool 空時は enemy_pool |

**mourngate elite_pool:** clock_moth, crystal_hedgehog（`enemy_type = ELITE`）

**報酬（通常戦闘より高い）:**

| 優先 | 報酬 | 実装 |
|---|---|---|
| 1 | Gold | EXP/Gold x1.5（`ELITE_REWARD_MULTIPLIER`） |
| 2 | Armor | 35% で leather_armor 即時ドロップ |
| 3 | Accessory | 25% で silver_ring 即時ドロップ |
| 4 | 高品質素材 | 15% `elite_relic_shard` x1 → `material_inventory` |
| 5 | EXP | Gold と同倍率 x1.5 |

**戦闘:** 既存自動戦闘（CombatTimer）。ボス固有 mechanics なし。

**UI:** 入室ログ `【エリート】{敵名}`、撃破ログに `(x1.5)` とボーナス行

## Discovery System（P2-Task018 確定）

最小限の**発見登録**システム。Codex UI なし。ログで新規発見を確認可能。

### ストレージ

| 項目 | 値 |
|---|---|
| 保存先 | `GameState.discovery_registry` |
| 形式 | `{ "category:entry_id": true }` |
| 永続化 | `SaveManager`（`discovery_registry` キー） |

### カテゴリ

| category | 登録タイミング | entry_id 例 |
|---|---|---|
| room | Special Room 入室 | heal / treasure / merchant / event / elite |
| enemy | 戦闘開始（敵遭遇） | sepia_hound / clock_moth |
| event | Event Room 入室 | fallen_altar / ancient_tome |
| lore | Event 選択（lore outcome） | ancient_record（mourngate イベント） |
| material | Event/Elite 選択（material outcome） | relic_shard / elite_relic_shard |

### ログ

初回登録時: `【新規発見】{category} / {entry_id}` を LabelLog に追記。

### 進行度メーター（既存・別系統）

`GameState.dungeon_progress[dungeon_id].discovery`（float 0.0〜1.0）は部屋入室毎に増加。
`discovery_unlocks` 閾値（hidden_room / hidden_boss）は将来コンテンツ用。**M3 では UI なし**。
