## 自動戦闘実装（MVP）

### CombatTimer

DungeonScene に配置した `Timer` ノード。

| プロパティ | 値 |
|---|---|
| wait_time | 1.5秒 |
| one_shot | false |
| autostart | false |

戦闘部屋入室時に `CombatTimer.start()` を呼び出す。敵撃破・全滅・探索終了時に `CombatTimer.stop()` を呼び出す。

### 1ティックの処理順序

```
CombatTimer.timeout
  ↓
SkillExecutor.tick(COMBAT_TICK_INTERVAL)   ← cooldown 減算
  ↓
_process_status_ticks()                    ← DoT / 状態ティック
  ↓
does_enemy_act_first()?
  → YES: _run_enemy_combat_phase() → is_party_wiped()? → _run_party_combat_phase()
  → NO : _run_party_combat_phase() → is_enemy_defeated()? → _run_enemy_combat_phase()
  ↓
_update_hp_bars() / _update_status_labels()
  ↓
is_party_wiped()? → YES: _handle_party_wipe()
```

`_run_party_combat_phase()` = `_do_party_attack()` + `_try_apply_skill_status()`  
`_run_enemy_combat_phase()` = skip 判定 or `_do_enemy_attack()`

## 冒険者HP管理（MVP実装）

### 管理クラス

`CombatController` がパーティHP全体を管理する（P2-D009）。

```gdscript
var party_combat_hp: Array[int] = []   # 現在HP（戦闘中のみ有効）
var party_max_hp: Array[int] = []      # 最大HP
const BASE_MEMBER_HP: int = 800        # base_stats.hp = 0 時のフォールバック（P3-STAT-CHAR / STAT_SCALE）
```

### 初期化タイミング

`start_combat(enemy_data)` 呼び出し時に `_init_party_hp()` で `GameState.party_members` から3人分を初期化する。

```
max_hp = member.base_stats.hp  （> 0 の場合）
       | BASE_MEMBER_HP (800)  （= 0 の場合のフォールバック）
```

### HP参照

DungeonSceneは `$CombatController.party_combat_hp[i]` と `GameState.party_members[i].display_name` を組み合わせて `LabelPlayerHp` に表示する。

## 死亡判定・全滅判定

### 死亡判定（個人）

```gdscript
func is_member_alive(index: int) -> bool:
    return party_combat_hp[index] > 0
```

- 死亡メンバーは `_do_party_attack()` でスキップ
- 死亡メンバーは敵の攻撃対象リストから除外
- LabelPlayerHp に `[戦死]` を表示
- 1人・2人の死亡では探索継続

### 全滅判定（パーティ）

```gdscript
func is_party_wiped() -> bool:
    return get_alive_count() == 0
```

### 全滅時の処理

```
CombatTimer.stop()
CombatController.end_combat()
ButtonNextRoom: disabled
ButtonFinish: disabled
LabelLog: "全員が倒れた... 探索失敗"
GameState.last_run_* ← 蓄積済み報酬を設定（ドロップは発生しない）
await 2.0秒
SceneRouter → ResultScene
```

## ダメージ計算（MVP実装）

### プレイヤー攻撃

```
攻撃ダメージ = base_damage × クリティカル補正

base_damage = weapon.rolled_attack + accessory.attack_bonus
            + AffixStatCalculator.attack_flat   # P2-Task031
            | FALLBACK_ATTACK (10)  ← 武器未装備時

crit_rate = weapon.critical_rate + accessory.crit_rate_bonus
          + AffixStatCalculator.crit_rate_add   # P2-Task031

クリティカル補正 = CRITICAL_MULTIPLIER (1.5)  ← クリティカル時
               | 1.0                          ← 通常時
```

- **生存メンバー全員**が同一の装備パラメータで攻撃する（MVP割り切り、P2-D011）
- 各メンバーは独立したクリティカル判定を行う
- 全メンバーのダメージ合計をLabelLogに表示

### 敵攻撃

```
最終ダメージ = max(1, round(base_dmg × K/(K+total_defense)))

base_dmg = enemy_data.attack × power（スキル等）
total_defense = (armor + accessory + affix + base_stats.defense) × job_def_mult
K = BalanceConfig.DEFENSE_MITIGATION_K（＝800）
```

- 敵は`alive`配列から**簡易ヘイト**（Guardian > Warrior > ランダム）で1人を選ぶ（P3-D024b）
- 味方→敵と同式の逓減軽減（P3-BAL-OPENING-002）。旧 flat `ATK−DEF` は廃止
- 最低1ダメージ保証（P2-D002）

## SkillExecutor（P2-Task025 確定）

| 項目 | 状態 |
|---|---|
| SkillData Resource | **定義済**（`scripts/data/SkillData.gd`） |
| サンプル | `resources/skills/slash_attack.tres`（power_multiplier 1.5 / cooldown 3.0s） |
| DataRegistry | `get_skill_data(id)` |
| SkillExecutor | **実装済**（`scripts/combat/SkillExecutor.gd`） |
| 戦闘接続 | **装備武器 fixed_skill_id → SkillData**（P2-Task026） |

### スキル解決フロー

```
GameState.get_member_equipped_weapon(member_index).weapon_id
  → DataRegistry.get_weapon_data(id).fixed_skill_id
  → DataRegistry.get_skill_data(skill_id)
  → SkillExecutor.execute_damage_skill()
  （fixed_skill_id 空 / SkillData 未取得時 → Constants.DEFAULT_PLAYER_SKILL_ID）
```

### ダメージ計算

```
skill_damage = base_damage × power_multiplier × クリティカル補正 × run_damage_multiplier

base_damage = weapon.rolled_attack + accessory.attack_bonus | FALLBACK_ATTACK (10)
```

- 通常攻撃と同一の base_damage / crit 判定を使用
- `power_multiplier` により通常 1 ヒットより高ダメージ（slash_attack = 1.5x）

### Cooldown

- `SkillExecutor.tick(COMBAT_TICK_INTERVAL)` を CombatTimer 毎に呼ぶ
- `COMBAT_TICK_INTERVAL` = 1.5s（Constants）
- slash_attack cooldown = 3.0s → 約 2 ティック毎に発動

### 戦闘ログ

```
攻撃: 30ダメージ
【スキル】鉄の剣 / 斬撃: 45ダメージ
```

（武器未装備時は `【スキル】斬撃: …` — slash_attack フォールバック）

| stat_type | 反映 |
|---|---|

**未対応（無視）:** Skill Power, Cooldown 等（Attack Speed はイニシアチブスコアに接続済み — P3-AFFIX-SPD-001）

## AI設定

### Combat Vision（目標）

ターゲットは AI が決定。優先順位はジョブ・武器によって変化する（`26_CombatVision.md`）。

| 候補 | 備考 |
|---|---|
| 最も近い敵 | Warrior 系 |
| 危険な敵 | Guardian |
| 敵後衛 | Scout |
| HP が低い敵 | 仕上げ |
| Elite / Boss | 脅威優先 |

### 現行 Alpha 実装

| 対象 | ルール |
|---|---|
| 味方攻撃 | 生存メンバー全員が同一ターゲット（敵1体） |
| 敵攻撃 | 生存メンバーから **簡易ヘイト** で1人を選択（優先: Guardian > Warrior > ランダム）（P3-D024b） |
| 優先度 | 最寄り（暗黙）・ボス部屋はボス固定 |

Guardian 在籍時は攻撃ログに `[守護]` プレフィックスを付与。Guardian 戦死後は次優先（Warrior > ランダム）に自動降格。

**过渡期ギャップ:** Scout 等のジョブ行動思想（DoT 特化等）はターゲット AI に未反映。→ **P3-D019 イニシアチブ** で拡張予定。

正式版でジョブ別ターゲット AI へ拡張する。

### イニシアチブ（P3-D019 Phase 1/2 + Affix — 実装済）

各戦闘開始時に先制判定を行う。

| スコア | 計算式 |
|---|---|
| パーティ | `max(weapon_inst.attack_speed × JobData.base_initiative_modifier × (1 + affix_attack_speed_mult_add))`（生存メンバー。武器なし→1.0、Affix なし→+0） |
| 敵 | `EnemyData.attack_speed`（未設定 → 1.0） |

- 敵スコア > パーティスコア → 敵先制（`[先制:敵]` ログ）
- それ以外 → パーティ先制（`[先制:パーティ]` ログ）

実装: `CombatController.get_party_initiative_score()` / `get_enemy_initiative_score()` / `does_enemy_act_first()` / `DungeonScene._run_party_combat_phase()` / `_run_enemy_combat_phase()`

| 段階 | 内容 | 状態 |
|---|---|---|
| Phase 1 | 武器 `weapon_inst.attack_speed` で行動順決定 | ✅ P3-INIT-001 |
| Phase 2 | ジョブ速度補正（`JobData.base_initiative_modifier`） | ✅ P3-INIT-002 |
| Phase 2.5 | Affix 速度補正（`attack_speed_mult_add`） | ✅ P3-AFFIX-SPD-001 |
| Phase 3 | Front/Mid/Back 位置 AI と統合 | Backlog |
