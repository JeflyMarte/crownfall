## P0 — プレイヤー役割：準備専用（P3-D024a）

### Alpha 現行（正）

プレイヤーの意思決定は **拠点・帰還後** に集中する。

- パーティ編成（メンバー別装備）
- 識別・装備更新
- ダンジョン選択
- 探索開始 / 速度 / 探索終了

ダンジョン中は **観察が主**。戦闘操作・移動指示は行わない。

### 正式版 Backlog（Phase 2）

最小のラン中介入を **1 ボタンから** 段階導入する。

| 方針（案） | 効果（案） |
|---|---|
| 集中火力 | ボス / エリート優先（ターゲット AI 軽微変更） |
| 安定 | 被ダメ軽減・撤退しやすさ |
| 撤退 | 次部屋以降の帰還ボーナス or HP 温存 |

**旧 spec の「探索優先 / 戦闘優先 / 収集優先 / 帰還優先」4 種は本 Backlog に統合。** 実装前に Task + Decision で数値確定。

## P1 — 戦闘：ロール可視化

### 簡易ヘイト（P3-D024b）

Alpha の次の戦闘改善。**Front / Guardian 優先被弾** を先に入れる。

| 段階 | 内容 |
|---|---|
| 短期 | 敵単体攻撃 → Guardian 在籍時は Guardian 優先（簡易ヘイト） |
| 中期 | ジョブ別スキル優先度（Scout = DoT 特化等） |
| 長期 | イニシアチブ + 位置（P3-D019 / P3-D019a） |

現行 Alpha（集中攻撃・ランダム被弾）は **过渡期** と明記。`08_戦闘_AI.md` 参照。

### 武器 stat 命名（P3-D024c）

| 現行フィールド | 方針 |
|---|---|
| `stun_power` | 状態異常 `stun` と混同する。**接続 Task 時に `stagger_power` へリネーム** |
| 未接続 stat | UI に出さない（`07_武器_装備.md` 接続表を正） |

## P1 — 属性・状態異常

### 学習コスト（P3-D024i）

初見プレイヤー向けに **Codex またはチュートリアル 1 画面** を必須とする。

- **属性** — 弱点時のみ ×1.25。無属性は通常ダメージ
- **状態異常** — DoT / 行動阻害等。**炎属性 ≠ 炎上**

### 弱点設計（P3-D024g）

弱点倍率 ×1.25 は維持。**弱点持ち敵の割合を上げすぎない**（全員属性武器必須化を避ける）。

### 呪い（P3-D024d）

| 対象 | Alpha〜Beta |
|---|---|
| プレイヤー → 敵 | 与ダメ ×0.75（現行） |
| 敵 → プレイヤー | **エリート以降 or 状態 UI 整備後** に限定。それまでは敵 on_hit curse を抑制 |
