## 武器パラメータ（MVP実装）

### WeaponData（マスターデータ）

| フィールド | 型 | 説明 |
|---|---|---|
| id | String | 武器ID（例: iron_sword） |
| display_name | String | 表示名 |
| fixed_skill_id | String | 固有スキルID（空なら DEFAULT_PLAYER_SKILL_ID） |
| base_attack | int | 基礎攻撃力 |
| rarity | int | Rarity enum値 |
| base_attack_speed | float | 基礎攻撃速度（1.0 = 標準） |
| base_critical_rate | float | クリティカル率（例: 0.05 = 5%） |
| base_knockback | float | ノックバック強度 |
| base_stagger_power | float | よろけ強度（**未接続・UI 非表示 — P3-D024c-001 リネーム済**） |
| base_attack_range | float | 攻撃範囲 |
| weight | float | 重量 |

### WeaponInstance（個体データ）

| フィールド | 型 | 説明 |
|---|---|---|
| instance_id | String | 個体識別ID（Time + randi） |
| weapon_id | String | 参照するWeaponDataのID |
| is_appraised | bool | 識別済みフラグ |
| rolled_attack | int | ロール済み攻撃力（base_attack + randi() % 6） |
| attack_speed | float | ロール済み攻撃速度 |
| critical_rate | float | ロール済みクリティカル率 |
| knockback | float | ロール済みノックバック |
| stagger_power | float | ロール済みよろけ強度（**未接続・UI 非表示 — P3-D024c-001**） |
| attack_range | float | ロール済み攻撃範囲 |
| weight | float | 重量 |

## AffixData（P2-Task028 — データ定義のみ）

Affix Bible Completed v1.0 に基づく Affix マスタ。Design Reference: `Affix_Bible_Completed_v1.0.md`

### 装備 Affix 容量（MVP 固定）

| 装備種 | Prefix | Suffix |
|---|---|---|
| Weapon | ×1 | ×1 |
| Armor | ×1 | — |
| Accessory | ×1 | — |

### AffixData フィールド

| フィールド | 説明 |
|---|---|
| id / display_name / description | 識別・表示 |
| affix_category | prefix / suffix |
| rarity | Enums.Rarity |
| stat_type | Attack, Defense, HP, Critical 等（Bible §6） |
| value | サンプル固定値 |
| tags | weapon / armor / accessory 等 |

### 接続状況

| 項目 | 状態 |
|---|---|
| AffixData Resource | **定義済** |
| DataRegistry.get_affix_data | **lookup** |
| AffixRoller | **実装済**（P2-Task029） |
| 識別 → Affix Roll | **接続済**（P2-Task030） |
| Instance prefix_ids / suffix_ids | **保存済**（P2-Task030 + Save） |
| Roll → 戦闘 stat | **接続済**（P2-Task031 — AffixStatCalculator） |
| Equipment Affix 表示 | **接続済**（P2-Task032 — AffixDisplayFormatter） |

### AffixRoller 利用

```gdscript
# 識別時（AppraisalController 内部）
AffixRoller.roll_for_equipment("weapon", item_rarity)  # prefix + suffix
AffixRoller.roll_for_equipment("armor", item_rarity)   # prefix only
AffixRoller.roll_for_equipment("accessory", item_rarity)
```

### 識別 Reveal 例

```
識別完了: rusted_blade  ATK 12
【Affix】鋭利 / 偉力
```

### Equipment UI 表示（P2-Task032）

識別済み装備のみ Affix 名・効果を表示。未識別は非表示。

```
rusted_blade  ATK 12  SPD 1.0  CRT 0%
Affix: 鋭利 / 偉力
Attack +3 / Attack +2
```

## 防具システム（MVP実装）

### ArmorData（マスターデータ）

| フィールド | 型 | 説明 |
|---|---|---|
| armor_id | String | 防具ID |
| display_name | String | 表示名 |
| base_defense | int | 基礎防御力 |
| base_hp_bonus | int | 基礎HP加算値 |
| base_resistance | float | 基礎耐性（0〜1.0） |
| weight | float | 重量 |
| rarity | int | Rarity enum値 |

### ArmorInstance（個体データ）

| フィールド | 型 | 説明 |
|---|---|---|
| instance_id | String | 個体識別ID |
| armor_id | String | 参照するArmorDataのID |
| rolled_defense | int | ロール済み防御力（base_defense + randi() % 4） |
| hp_bonus | int | HP加算値 |
| resistance | float | 耐性値 |
| weight | float | 重量 |
| rarity | int | レアリティ |
| is_appraised | bool | 識別済みフラグ |

### 防具ドロップ

- ドロップ率: **30%**（ダンジョン終了時）
- ドロップ元: `DungeonController.generate_run_loot()` → `_generate_armor_loot()`
- 保存先: `GameState.armor_inventory`

### 防具識別

- 識別コスト: **100G**（武器・防具・装飾品共通）
- 識別後: `is_appraised = true`、識別ログ表示
- 識別ロジック: `AppraisalController.appraise_next()`（duck typing: `"armor_id" in item` で識別）

### 防具装備

- 条件: `is_appraised == true`
- 装備操作: `EquipmentController.equip_armor(item, member_index)` → `Adventurer.equipped_armor`
- 1 instance = 1 メンバーのみ（P3-D018）
- 効果反映: メンバー index ごとの戦闘 DEF / HP

### MVP戦闘での接続状況

| パラメータ | 接続状況 |
|---|---|
| rolled_defense | **接続済み**（敵ダメージ軽減に使用） |
| hp_bonus | **接続済み**（per-member 最大 HP） |
| resistance | 未接続（将来実装） |

## 装飾品システム（Accessory_Bible_v1.0）

### スロット数

3（**メンバー各1枠** — weapon / armor / accessory。P3-D018）

### 基本パラメータ

| パラメータ | 説明 | 実装状況 |
|---|---|---|
| Critical Bonus | クリティカル率加算値 | **MVP実装済み** |
| Gold Bonus | ゴールドドロップ倍率 | 将来実装 |
| Luck | 幸運値（影響範囲は下記参照） | 将来実装 |
| Movement Speed | 移動速度加算値 | 将来実装 |
| HP Recovery | HP回復量加算値 | 将来実装 |
| Experience Bonus | 経験値ボーナス倍率 | 将来実装 |

### Luckの効果範囲

Luck が影響するのは以下のみ：

- Rare Drop（レアドロップ率）
- Treasure Quality（宝箱の品質）
- Event Success（イベント成功率）
- 将来拡張予定の項目

**Luck はクリティカル率（Critical Bonus）には影響しない。**

### 入手経路

優先順位順：

1. Dungeon Reward（ダンジョン報酬）
2. Treasure Room（宝箱ルーム）
3. Boss Reward（ボス報酬）
4. Merchant Room（ダンジョン内商人 — 防具・装飾品・回復薬。**武器は販売しない**）

### Dungeon Merchant Room（P2-Task015）

探索中の Merchant Room で購入可能（Branch Route 経由）。

| 種別 | 商品 | 価格 | 付与 |
|---|---|---|---|
| 防具 | 革鎧 | 40G | `ArmorInstance`（未識別）→ `armor_inventory` |
| 装飾品 | 銀の指輪 | 60G | `AccessoryInstance`（未識別）→ `accessory_inventory` |
| 回復 | 回復薬 | 35G | 生存メンバー +15 HP（即時） |

- 支払い: 永続 `GameState.gold`
- 1 商品 1 回のみ購入可（`purchased` フラグ）
- 武器は商人カタログに含めない（P2-D020）
- 素材（Materials）: MaterialData 未実装のため将来 Task

## Save/Load（MVP実装）

SaveManagerが `user://save_data.json` にJSON形式で保存する。

### 保存対象

| キー | 内容 |
|---|---|
| gold | 所持Gold |
| party | party_members の配列（id / display_name / level / job_id / base_stats） |
| inventory | WeaponInstanceの配列（全フィールド） |
| equipment.weapon | 装備WeaponInstanceのinstance_id |
| armor_inventory | ArmorInstanceの配列（全フィールド） |
| equipped_armor.armor | 装備ArmorInstanceのinstance_id |
| accessory_inventory | AccessoryInstanceの配列 |
| equipped_accessory.accessory | 装備AccessoryInstanceのinstance_id |
| dungeon_progress | ダンジョン別発見度辞書 |

### 復元ルール

- 装備はinstance_idで保存し、起動時にinventoryを検索して参照を復元（P2-D010・D011）
- inventory復元 → equipment復元 の順序を強制（P2-D011）
- セーブ破損時はGameStateのデフォルト値（party生成・Gold=0）で起動

### セーブタイミング

- `AppraisalController.appraise_next()` — 識別完了後
- `EquipmentController.equip_weapon/armor/accessory()` — 装備変更後
- `AppraisalScene._on_back_pressed()` — 識別画面から戻るとき
- `EquipmentScene._on_back_pressed()` — 装備画面から戻るとき
