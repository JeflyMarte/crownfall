# 01_ゲーム概要

## タイトル

Crownfall（仮）

## ジャンル

- スマホ向け
- 2D見下ろし型RPG
- 自動探索
- 自動戦闘
- ハクスラ
- ローグライト要素あり

## コアコンセプト

プレイヤーは冒険者を直接操作しない。
探索隊の指揮官として、方針・装備・編成・ビルドを決め、**戦闘は観察する**（`26_CombatVision.md`）。

探索・戦闘はシームレスに進行し、戦闘専用画面はない（長期目標）。現行 Alpha は部屋単位ステップ + CombatTimer 抽象戦闘。

## コア体験

- 遺跡を選ぶ
- 3人パーティを編成する（メンバー別装備）
- 自動探索を見守る（**戦闘は観察** — P3-D024a）
- 武器を拾う
- 拠点で鑑定する
- 装備更新する
- さらに深い遺跡へ向かう

**Alpha 現行:** 意思決定は拠点・帰還後が中心。ダンジョン中の方針切替は Backlog（`28_ゲームデザイン点検.md`）。

## ゲームの勝ち筋

単に強い武器を拾うゲームではなく、強い組み合わせを作るゲーム。

ビルド例（現行体系 — `27_状態異常と属性.md`）

- 状態異常（毒 / 炎上 / 感電 / 冷却 等）
- 属性弱点（5 属性 × ×1.25）
- クリティカル
- 探索・発見（Codex）
- 防御・生存（将来: ヘイト / イニシアチブ）

MVP 検証ビルド（P3-D024h）: **状態異常・属性弱点・クリティカル**。召喚・旧出血体系は Backlog。
