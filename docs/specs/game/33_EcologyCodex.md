## コンセプト

Ecology Codex は、探索者ギルドが長年蓄積してきた調査記録である。プレイヤーは**討伐数ではなく、調査の進捗**によって図鑑を完成させていく。

図鑑は「世界観の説明 / 探索の達成感 / モンスターの理解 / 攻略ヒント」を一つのシステムで担う。

## モンスター詳細

| 区分 | 項目 |
|---|---|
| 基本情報 | 名称 / 分類 / 危険度 / 生息地 / 初回発見地域 |
| 生態 | 食性 / 活動時間 / 縄張り / 群れの規模 / 特徴 |
| 戦闘情報 | 弱点 / 耐性 / 行動傾向（※ゲーム内で判明した情報のみ表示） |
| 採取素材 | 通常素材 / 希少素材 / 用途 |
| 調査記録 | 探索者ギルドが残した短い文章（100〜200 文字程度。余白を残す） |

調査記録の例:
> 崩れた地下水路では単独行動が多い。光を極端に嫌い、水辺で獲物を待ち伏せする。

---

## 調査進行（5 段階）

| Stage | 名称 | 開放内容 |
|---|---|---|
| 1 | ？？？？？ | シルエットのみ |
| 2 | 発見 | 名前と外見が判明 |
| 3 | 初回討伐 | 基本情報が開放 |
| 4 | 追加調査 | 生態情報が追加 |
| 5 | 調査完了 | 素材・戦闘情報・調査記録がすべて閲覧可能 |

図鑑は一度倒しただけでは完成しない。

---

## ボス図鑑

頂点捕食者は専用ページを持つ。追加項目:

- 生態系での役割 / 推定寿命 / 縄張り範囲 / 発見記録

一般種より詳細な記録が残る。

---

## 世界観との接続

探索者は討伐数を競う組織ではない。**新しい発見ほど高く評価される**（未知の生物を発見 / 新素材を確認 / 新しい行動を観察）。

---

## MVP 範囲

MVP ではシンプルな構成とする。

- モンスター一覧 / 名前 / イラスト / 生息地 / 素材 / 調査記録

食性や活動時間などの詳細データは将来拡張する。

## Future（正式版）

生態ツリー / 地域ごとの分布図 / 季節による出現変化 / 希少種発見率 / サイズ記録 / 個体差 / 生息数推定

---

## Design Decision

「図鑑を 100% にすること」ではなく「世界を理解していくこと」を目的に設計する。コンプリート前提の膨大な数値埋めではなく、プレイヤーが「この生物はこういう暮らしをしているのか」と感じられる内容を重視する。
