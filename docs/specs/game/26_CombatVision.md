# 26_CombatVision — 戦闘ビジョン

**Status:** SSOT（DevelopmentHQ 承認済 — P2-D166）  
**Version:** v1.0  
**Approved:** 2026-06-23  
**Invariant:** 本書の Core Concept / Player Role / Development Rule は長期的に不変とする

---

## Purpose

Crownfall の戦闘は、プレイヤーがキャラクターを直接操作するゲームではない。

プレイヤーは**探索隊の指揮官**として、以下を構築し、その結果を**観察・改善**する。

- パーティ編成
- ジョブ
- 武器
- 装備
- Affix
- ビルド

戦闘は「操作の場」ではなく、**準備の成果を可視化する時間**である。

---

## Core Concept

探索・移動・戦闘はすべて**シームレス**に進行する。

- 戦闘専用画面への切り替えは行わない
- ダンジョン内ではドットキャラクターがリアルタイムで自律行動する
- 敵発見 → 接敵 → 戦闘 → 撃破 → 探索再開が一連の流れとして処理される

```text
探索
  ↓
敵発見
  ↓
自然に交戦
  ↓
撃破
  ↓
探索再開
```

このテンポを維持する。

---

## Player Role

戦闘中、プレイヤーは**操作しない**。

操作対象は戦闘ではなく、拠点・装備画面における以下である。

| 対象 | 役割 |
|---|---|
| 装備 | 防御・補助・Affix |
| 武器 | 射程・火力・スキル identity |
| パーティ | 編成・役割分担 |
| ジョブ | AI 行動思想 |
| ビルド | 上記の組み合わせ |

---

## AI Behavior

味方・敵ともに AI で行動する。

各ユニットは以下を考慮して行動する。

- 現在位置
- 武器射程（理想距離）
- ジョブ思想
- ターゲット優先度

プレイヤーは**移動指示を出さない**。

---

## Position System

位置は戦術の一部である。戦闘中の隊列は固定ではなく、AI によって自然に変化する。

| 隊列 | 役割 |
|---|---|
| **Front** | 敵を受け止める |
| **Mid** | 前衛を支援する |
| **Back** | 安全距離を維持する |

---

## Weapon Identity

武器種によって**理想距離**と**立ち回り**が異なる。

| 武器種（Vision） | ゲーム内対応 | 理想距離 | 特徴 |
|---|---|---|---|
| Dagger（短剣） | 双剣（将来: 短剣種） | 近距離 | 高速接近・単体火力 |
| Greatsword（大剣） | 大剣 | 近距離 | 低攻撃速度・高火力 |
| Spear（槍） | 槍 | 中距離 | 前衛維持・安定性 |
| Bow（弓） | 弓 | 遠距離 | 後衛維持・継続火力 |
| Staff（杖） | 杖 | 後衛 | 広範囲・支援・魔法 |
| Shield（盾） | 将来（副手 / 防具連動） | 近距離 | 敵引き付け・味方防御 |

AI は常に武器の理想距離を維持しようとする。

| 例 | 行動 |
|---|---|
| 弓 | 敵へ近付きすぎない |
| 槍 | 少し距離を空ける |
| 短剣 | 積極的に接近する |

`WeaponData.attack_range` は将来、この理想距離 AI の入力となる（現行 Alpha 未接続）。

---

## Job Identity

ジョブはスキルリストではなく、**AI の行動思想**を決定する。

武器とジョブを組み合わせることで、同じ武器でも立ち回りが変化する。

| ジョブ（Vision） | 実装 id | AI 思想 |
|---|---|---|
| Warrior | `warrior` | 最も近い敵へ接近 |
| Guardian | `guardian` | 味方を守る位置を維持 |
| Scout | `scout` | 敵後衛を狙う |
| Hunter | 将来 | 距離を維持して攻撃 |
| Mage | 将来 | 敵集団を優先 |

詳細: `06_キャラクター_ジョブ.md` / `JobData.passive_tag_ids`

---

## Target Selection

ターゲットは AI が決定する。優先順位はジョブによって変化する。

| 候補 | 例 |
|---|---|
| 最も近い敵 | Warrior デフォルト |
| 危険な敵 | Guardian / Scout |
| 敵後衛 | Scout |
| HP が低い敵 | 仕上げ優先 |
| Elite | 脅威優先 |
| Boss | ボス優先 |

現行 Alpha（MVP）: 最寄り + ボス優先のみ。上記は正式版ロードマップ。

---

## Combat Presentation

重要なのは**「見るだけで理解できること」**。

派手な演出より、以下が分かることを優先する。

- **誰が**
- **誰を**
- **何で**
- **なぜ**（射程・隊列・ジョブ）

攻撃しているのか。

---

## Exploration Integration

探索中もキャラクターは歩き続ける。

- 敵を発見すると自然に接近する
- 戦闘終了後はそのまま探索へ戻る
- ロードや戦闘画面への遷移は行わない

### 現行 Alpha との関係

| 層 | Combat Vision | 現行実装 |
|---|---|---|
| マクロ進行 | シームレス探索（目標） | 部屋単位ステップ（`01_MVP方針決定` §3） |
| 戦闘表現 | リアルタイム自律ドット（目標） | CombatTimer 1.5s 抽象戦闘 |
| 画面遷移 | 戦闘専用画面なし | DungeonScene 内で完結（遷移なし ✓） |

マクロの部屋ステップは**过渡期の実装**。Combat Vision は戦闘・プレゼンテーション層の長期正（P2-D168）。

---

## Design Goal

理想のプレイ体験:

> **「見るだけなのに面白い」**

プレイヤーが記憶に残すのは操作ではなく、以下の体験である。

- この武器が強かった
- この編成が機能した
- このビルドが面白かった
- このジョブ構成が噛み合った

---

## Future Extensions

Combat Vision を**変更せず**追加可能な拡張要素。

| カテゴリ | 例 |
|---|---|
| 物理 | ノックバック、回避、ガード、カウンター |
| 状態 | 状態異常、範囲攻撃 |
| 空間 | 飛行ユニット、地形効果、高低差 |
| 戦術 | ヘイト制御、包囲 |
| ボス | 大型ボス、部位破壊 |
| 環境 | 天候 |

これらは Vision を実現するための拡張であり、Vision 自体の置き換えではない。

---

## Development Rule

**Combat Vision は長期的に不変とする。**

新システムを追加する場合も、以下を変更してはならない。

> プレイヤーは指揮官であり、AI が探索・戦闘を自律して行う。
