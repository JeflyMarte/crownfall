## UI_Reference_003_07_Battle_Auto_v2 — 実装スコープ（P3-D011）

**Decision:** OD-UI-001 戦闘画面 — 段階的寄せ（B）の第 1 段。**手動戦闘は実装しない**（Combat Vision 維持）。

### IN（Phase UI-1）

| 領域 | 内容 |
|---|---|
| ヘッダー | ダンジョン名・部屋 n/m・部屋種別バッジ・x1/x2・一時停止（観察用） |
| バトルフィールド | CHR 3 + 敵スプライト・**頭上 HP バー**・VFX・ダメージ数字 |
| バトルログ | ScrollContainer・タイムスタンプ・複数行 |
| 自動戦闘行 | 「自動戦闘中」表示 |
| 非戦闘ゾーン | 分岐 / 商人 / イベント / 次の部屋 / 探索終了（戦闘中は非表示） |

### OUT（v2 で明示除外）

| 要素 | 理由 |
|---|---|
| タイマー / 獲得予定報酬 | オーナー判断 |
| ダンジョン中の下部 5 タブ | 拠点専用 |
| キャラカード列（下部 HP パネル） | 頭上 HP に集約 |
| 手動攻撃・スキル操作 | SSOT 外 |
| WAVE 表記 | 部屋 n/m に置換 |

### 実装前提（P3-D012）

| 項目 | 決定 |
|---|---|
| 画面比率 | 縦長スマホ固定 |
| RoomArt | バトルフィールド主体。64×64 タイルは背景化 or 非表示 |
| ≡ メニュー | 探索終了のみ |
| 倍速 | x1=1.5s / x2=0.75s。停止=Timer 停止（手動戦闘なし） |
| 背景 art | ChatGPT 生成可（P3-D013）。PA タイルは不要 |

### 将来（Phase UI-2+）— ✅ Closeout（2026-06-25）

| 項目 | Task |
|---|---|
| 状態異常アイコン | P3-UI2-013 |
| エリート/ボス演出フレーム | P3-UI2-014 |

## UI_Reference_001 — 複合ダッシュボード

**レイアウト:** 1280×720 想定の横画面。複数パネルを 1 枚に配置した「理想ダッシュボード」。

### パネル概要

| 領域 | モック内容 |
|---|---|
| 左上 | 拠点（王都アバロン）— メニュー多数・複数通貨 |
| 左中 | 冒険者編成 — 4 人・装備スロット |
| 左下 | 遺産の地図 — ノードマップ・発見率 |

| 右下 | 図鑑 — 敵 68/128・ドロップ表示 |
| フッター | 4 つの訴求（自動探索、ビルド、九王遺産、発見コレクション） |

**主用途:** Phase3-A 以降の方針相談（OD-UI-001 等）・Combat Vision 目標・拠点/探索/図鑑の情報密度の正。

## UI_Reference_002 — スマホ 7 画面

**レイアウト:** スマホ縦向き・1 画面 1 機能。各画面に「特徴」「ポイント」注釈付き。

| # | 画面 | モック内容 | 現行シーン |
|---|---|---|---|
| 1 | 探索画面 | 自動進行中・パーティ HP・ミニマップ・戦闘ビュー・帰還 | `DungeonScene` |
| 2 | 拠点画面 | 王都アバロン全景・左メニュー・下部タブナビ | `BaseScene` |
| 3 | 武器一覧 | カテゴリタブ・レアリティフィルタ・グリッド | （未実装 — Equipment 簡易） |
| 4 | 装備・パーティ | ステータス・大キャラ・装備スロット・4 人切替 | `EquipmentScene` |
| 5 | 鑑定画面 | レジェンド演出・固有効果・売却/保管/装備 | `AppraisalScene` |
| 6 | ダンジョン選択 | 遺跡タブ・推奨 Lv・発見率・探索ボタン | `BaseScene` DG 選択 |
| 7 | 探索リザルト | 時間/階層/撃破数・獲得物・発見率増加 | `ResultScene` |

**主用途:** スマホ実装時の画面遷移・タブ構成・1 画面あたりの情報量の正。001 の各パネルを「単独画面化」した版。

### 001 と 002 の関係

```
UI_Reference_001  … 全部載せた理想ダッシュボード（横画面・情報統合）
UI_Reference_002  … 同じ機能をスマホ縦・画面分割した版
```

方向性（自動探索・ビルド・九王遺産・発見コレクション）は同一。レイアウトと画面遷移の粒度が異なる。

## 採用（Adopted）

現行 SSOT・実装方針と一致。方向性として維持する。

| 要素 | モック | 現行 / SSOT |
|---|---|---|
| ジャンルコンセプト | 自動探索ハクスラ RPG | ✅ プロジェクト憲章 |
| 拠点 → 探索 → 結果ループ | Base → Dungeon → Result | ✅ 実装済 |
| 自動戦闘 | パーティ vs 敵の自動戦闘 | ✅ CombatTimer 自動戦闘 |
| 複数ダンジョン | 忘都回廊など | ✅ 2 DG（王都跡 / 白骸墓地） |
| 装備ビルド | 武器・防具・アクセサリ | ✅ M1 3 枠装備 |
| 鑑定 | 新武器発見演出 | ✅ AppraisalScene |
| Special Room | 探索中の多様な出来事 | ✅ Heal/Treasure/Merchant/Event/Elite/Branch |
| 発見・コレクション | 図鑑・発見率 | ⏳ Codex UI 済。発見率 UI は Beta |
| 九王・王遺産ナラティブ | Heritage of the Nine Kings | 📋 Lore / 正式版（王遺産枠は MVP 外） |
| レジェンド装備 | 夜王の断罪剣など | 📋 レアリティ設計あり。演出・固有 UI は Beta |
| ダンジョン名表示 | 忘都回廊 B7F | ✅ `DungeonData.display_name` + 部屋ラベル |
| 探索ログ | 戦闘・宝箱ログ | ✅ LabelLog |
| クリティカル等戦闘 feedback | クリティカル表示 | ✅ ダメージ計算・ログ |
| 下部タブナビ（002） | Home / Party / Explore / Bag / Shop | 📋 Beta UI 候補 |
| 武器一覧グリッド（002） | カテゴリ・レアリティ・一括売却 | 📋 Beta UI 候補 |

## 変更して採用（Adopted with Changes）

モックの意図は採用するが、MVP / Phase2 では **表現・規模を縮小** する。

| 要素 | モック（001/002） | 採用形（現行・予定） | 理由 |
|---|---|---|---|
| 探索進行 | リアルタイム横スクロール自動進行 | **部屋単位ステップ進行** | MVP 方針 D-003 確定 |
| 探索 UI | キャラ・敵スプライト、ミニマップ、倍速/一時停止/撤退 | **ラベル + 次の部屋ボタン** | MVP 簡易 UI。演出は Beta |
| 拠点 UI | 王都アバロン全景 + 7 メニュー | **最小 BaseScene**（Gold・装備・DG 選択） | 段階拡張 |
| パーティ人数 | 4 人編成 | **3 人**（Constants.MAX_PARTY_SIZE） | 現行実装 |
| 装備枠 | 4 スロット / 人 | **3 枠**（武器・防具・装飾品） | MVP 方針。4 枠目「王遺産」は正式版 |
| 通貨・素材表示 | Gold + 複数リソース | **Gold 中心**（鑑定・クラフト拡張中） | MVP 方針 D-005 |
| ダンジョン選択 | リスト + 発見率（002 §6） | **ボタン選択** → 将来リスト UI | M4 最小 UI から拡張 |
| 図鑑 | 68/128 敵・星評価・ドロップ表 | **CodexScene 最小 UI** | M9 基盤済。002 密度は Beta |
| 遺産の地図 | ノード + 発見率 | **`dungeon_progress.discovery` float** | 進行度は実装済。マップ UI 未実装 |
| 武器詳細 | レベル/50・強化・売却・固有効果 | **簡易 Instance + 鑑定** | Affix/強化は M5+ |
| 戦利品画面 | 専用リザルト + 一括鑑定（002 §7） | **ResultScene + Appraisal** | MVP フローで代替 |
| リザルト項目 | 時間・最深階・撃破数（002 §7） | **EXP / Gold / ドロップ簡易** | Beta で 002 項目拡張 |
