# 01 — ゲーム概要（現行）

## タイトル

Crownfall（仮）

## ジャンル・プラットフォーム

- スマホ向け（縦画面 **720×1280**、`aspect=keep`）
- 2D 見下ろし型 RPG
- **自動探索** ＋ **自動戦闘**
- ハクスラ（装備収集・ビルド）
- ローグライト要素あり（ラン単位の探索）
- エンジン: Godot 4.6.x / GDScript

## コアコンセプト

プレイヤーは冒険者を **直接操作しない**。  
探索隊の **指揮官**として、方針・装備・編成・ビルドを決め、**戦闘は観察する**。

目標体験: 「見るだけなのに面白い」（Combat Vision）。

## コア体験（ループの核）

1. 拠点で備え（編成・装備・鍛冶・招待状）
2. ダンジョン（章）を選ぶ
3. 自動探索を見守る（部屋ステップ進行）
4. 帰還して報酬・成長を確認
5. 装備を更新し、また出る

## 勝ち筋

「強い武器を拾う」だけでなく、**強い組み合わせを作る**ゲーム。

現行で検証している柱:

- 状態異常（毒／炎上／感電／冷却 等）
- 属性弱点（5属性 × 弱点時 ×1.25）
- クリティカル
- 探索・発見（図鑑／Codex）
- 防御・生存（簡易ヘイト・イニシアチブ）

## 公開βの体験範囲（重要）

| 項目 | 現行 |
|---|---|
| プレイ可能 | **① モーンゲート** ノーマル（1-1〜1-5） |
| ②〜⑤／寄り道 | データはあるが **選択不可（🔒）**。削除しない |
| Hard／Nightmare | 実装あり。β体験外（メイン5ノーマル全クリア後想定） |
| 助っ人招待状 | **常時 ON**（単発魔晶石＋チケット） |

## プレイヤーロール

- 隊長（指揮官）— 名前入力あり。調査許可ランク D〜S
- 冒険者パーティ — 人間枠＋オトモ
- NPC 案内 — 記録官 **ニーナ**（導入・拠点おすすめ）

## 何をしないか（設計境界）

- ラン中の細かい手動操作・リアルタイム位置操作は未実装（将来 Vision）
- 操作講習型チュートリアルは採用しない
- 鑑定屋への必須導線は現行オミット
