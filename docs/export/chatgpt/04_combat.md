# 04 — 戦闘・属性・スキル

## 方針

- **完全自動戦闘**。プレイヤーは攻撃操作しない
- 戦闘はダンジョンシーン内（専用戦闘画面なし）
- 進行は **CombatTimer** ベースの抽象戦闘（テキストログ＋スプライト演出）
- 長期目標はリアルタイム自律移動（Vision）。現行は過渡期

## パーティと全滅

- 1人死亡しても探索継続
- **人間パーティ全滅**で探索失敗
- 編成: 人間最大 **4** ＋ 随伴オトモ（枠外）

## ティック概要

1. スキルCD減算  
2. 状態異常ティック（DoT等）  
3. 先制判定（イニシアチブ）→ 味方／敵フェーズ  
4. HP／状態UI更新  
5. 全滅判定  

ダメージ最終段に **±10% 乱数**。

## 属性（Element）— 5種

`fire` / `ice` / `thunder` / `dark` / `holy`

| 条件 | 倍率 |
|---|---|
| 攻撃に属性あり **かつ** 敵弱点 | ×1.25 |
| それ以外 | ×1.0 |

- 耐性ペナルティ（×0.75）は **不採用**
- **炎属性 ≠ 炎上**。属性は弱点ダメのみ。状態は別経路

世界観上、属性は自然力「エルダ」の相（超自然魔法ではない）。

## 状態異常（Status）

DoT・行動阻害・与／被ダメ補正など時間軸の効果。  
VFX（付与バースト／オーラ／DoT tick）あり。  
詳細ID・数値は本体 `27_状態異常と属性.md`。

## スキル体系（現行）

| 種別 | 扱い |
|---|---|
| ジョブスキル | Lv到達で習得。**装備枠は1本** |
| 武器スキル | **レジェンド武器**の `fixed_skill_id`（装備枠外） |
| 必殺 | キャラ固有。カットイン演出あり |

### 必殺チャージ（現行）

- 与ダメージ／被ダメージでチャージ
- 下UIに技ゲージ＋必殺ゲージ

### 必殺演出

顔ノフレーム＋帯＋暗転など。ダメ／回復とも派手化済み（要実機確認）。

## 戦術（ガンビットUI）

- プリセット一行表示
- 「使う技／いつ使うか」等の行動ルール
- ラン中の細かい介入より、拠点での設定が中心

## AI・ヘイト・イニシアチブ

- 簡易ヘイト（タンク fantasy）
- イニシアチブで先制
- Affix の速度はイニシアチブへ接続
- イベント助っ人は敵の狙撃対象から除外（無敵タンク化防止）

## 戦闘UIの主な要素

- Now Playing 帯／戦術ログ／ターン順バッジ
- 浮動ダメージ数字／HPバー追従
- 状態異常頭上バッジ
- エリート／ボス枠、ボス WARNING、エリート登場演出
- 行動順の敵アイコン
